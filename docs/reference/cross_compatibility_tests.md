# Z98 Cross-Compatibility Tests

This document records the results of compiling the Z98 Stage 0 bootstrap compiler (`zig0`) with various C++98 compilers and verifying its ability to generate correct C89 code.

## Environment Summary
- **OS**: Linux (Ubuntu 24.04 via environment)
- **Architecture**: x86_64 (tests targeting i386/32-bit)
- **Tools installed**: `g++-multilib`, `clang`, `musl-tools`, `mingw-w64`, `wine`

## Test Case 3: MinGW (i686-w64-mingw32-g++)
- **Compiler**: `i686-w64-mingw32-g++ (GCC) 13.2.0`
- **Command**: `i686-w64-mingw32-g++ -std=c++98 -static -Isrc/include src/bootstrap/bootstrap_all.cpp -o zig0_mingw.exe`

### zig0 Compilation
- **Status**: PASS (after minor identifier renaming to avoid Windows header conflicts)
- **Warnings/Errors**:
- Initial failure due to `TOKEN_TYPE` and `PS_NULL` colliding with macros/enums in `windows.h`.
- Renamed `TOKEN_TYPE` to `TOKEN_TYPE_KEYWORD` and `PS_NULL` to `PS_IS_NULL`.
- Final compilation successful with baseline C90 warnings.

## Test Case 4: Musl (musl-gcc)
- **Details**: `musl-gcc` is a wrapper for C. Since `zig0` is C++98, it cannot be compiled directly with `musl-gcc`. However, the C89 code generated by `zig0` can be compiled and linked against `musl`.

## Summary of Fixes for Cross-Compatibility
To ensure compatibility with Windows headers (via MinGW), the following internal compiler identifiers were renamed:
1. `TOKEN_TYPE` -> `TOKEN_TYPE_KEYWORD` (conflicted with `TOKEN_TYPE` enum in `winnt.h`).
2. `PS_NULL` -> `PS_IS_NULL` (conflicted with `PS_NULL` macro in `wingdi.h`).
